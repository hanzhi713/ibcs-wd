11.4

The first output is :
Test 1: False

The second output is:
Test 2: True

At first, 'This' refers to the list assigned to it and so do 'That'. The content of the two lists are the same, but they are stored in different places in the computer. The 'is' operator is used to check if the two lists are the same list, or, say, if the two names on it's two sides are alias. Therefore the first test returns 'False'.
When we assign 'this' to 'that' in line 4, 'this' and 'that' become alias, or, say, they refer to the same list. Therefore the second test returns 'True'.

def cross_product(u,v):
    c=[]
    for i in range(len(u)):
        c.append(u[(i+1)%3]*v[(i+2)%3]-u[(i+2)%3]*v[(i+1)%3])
    return c
##>>> cross_product([1,1,1],[1,1,1]) == [0,0,0]
##True
##>>> cross_product([1,2,3],[4,5,6]) == [-3,6,-3]
##True
##>>> cross_product([1,2,3],[3,2,1]) == [-4,8,-4]
##True

11.11

before swap function call: a: ['This', 'is', 'fun'] b: [2, 3, 4]
before swap statement: x: ['This', 'is', 'fun'] y: [2, 3, 4]
after swap statement: x: [2, 3, 4] y: ['This', 'is', 'fun']
after swap function call: a: ['This', 'is', 'fun'] b: [2, 3, 4]

a,b are not swapped

Reason:
  • x,y are just local variables. NOT alias of a,b
  • The function isn't returning anything useful

So after the call 'swap', a is still ['This', 'is', 'fun'] and b is still [2, 3, 4]. We didn't make any change to them.

To fix this problem, we can add 'return (x,y)' at the end of the function.
