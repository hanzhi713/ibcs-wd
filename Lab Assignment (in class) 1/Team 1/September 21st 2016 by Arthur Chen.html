2.Lists


After the test, we talked about some list methods.

First, we discussed lists aliasing. We may encounter this problem when defining functions and assigning a list to a different name, like "b=a".
In a function definition, the parameters are aliases of the arguments. If we make changes to the parameters, the arguments also change, then we have a modifier. Otherwise, we have a pure function.
When we are assigning a to be, we are actually making a alias, b, of a. This means that b and a refer to the same object. If we make changes to b, a will change accordingly. If we want b to be a new independent list, we can use b=a[:], which has the same effect as deepcopy.

Then, we talked about nested lists. We usually use this to represent matrices. For example, lst=[[1,2,3],[2,3,4],[3,4,5]] is a 3*3 matrix. The elements in it are called by giving two indexes, like lst[2,2] for 5 as python counts from 0.

Also, we learnt that the function range() only generates a "lazy list", which means that there isn't actually a list, but a request for a list. This can make our program more efficient when we are finding the first element of some property in a large dataset. It is because we don't actually examine the rest elements as it is lazy.
